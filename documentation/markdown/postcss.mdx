Through building Obsidian.css, it felt necessary to extend PostCSS with more plugins for analysis. Honestly, would encourage anyone to do this, [the API](https://github.com/postcss/postcss/blob/master/docs/writing-a-plugin.md) is well documented and there are dozens of examples to follow.

### PostCSS Parker

Parker is a CSS analysis tool to gives fine-grained stats about your stylesheet. This is a wrapper around [Parker](https://github.com/katiefenn/parker) to be used in a PostCSS pipeline and output to an named file.

### PostCSS Gzip

Simple output file size reporter.

### [PostCSS MDGX](https://github.com/charliewilco/postcss-mdgx)

Fork of [postcss-style-guide](https://github.com/morishitter/postcss-style-guide) to tweak the output of the markdown parser. Archived.

### [Wilco](https://github.com/charliewilco/wilco)

Wilco is an opinionated linter based off of stylelint. Wilco is an opinionated linter for CSS that's powered by Stylelint. It's the first command line app I've ever started and finished. The focus is meant to just take in a glob of files and check it against a specific rule set for style violations.
